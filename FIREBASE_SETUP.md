# Firebase Setup Guide for Society Guardian

## Problem
Your app is missing Firebase configuration. The Firestore database doesn't exist yet because Firebase hasn't been set up.

## Solution: Step-by-Step Setup

### Step 1: Create a Firebase Project
1. Go to https://console.firebase.google.com/
2. Click "Add project" or "Create a project"
3. Enter a project name (e.g., "society-guardian")
4. Follow the setup wizard (you can disable Google Analytics if you want)
5. Click "Create project"

### Step 2: Register Your Flutter App
1. In Firebase Console, click the Android icon (🤖)
2. Enter your Android package name: `com.example.society_guardian` (check your `android/app/build.gradle` for the actual `applicationId`)
3. Download the `google-services.json` file
4. Place it in: `/workspace/android/app/google-services.json`

5. Click the iOS icon (🍎)
6. Enter your iOS bundle ID: `com.example.societyGuardian` (check your `ios/Runner/Info.plist`)
7. Download the `GoogleService-Info.plist` file
8. Place it in: `/workspace/ios/Runner/GoogleService-Info.plist`

### Step 3: Install FlutterFire CLI
```bash
dart pub global activate flutterfire_cli
```

### Step 4: Configure Firebase Automatically
Run this command in your project root:
```bash
flutterfire configure --project=YOUR_PROJECT_ID
```
Replace `YOUR_PROJECT_ID` with your actual Firebase project ID.

This will automatically:
- Generate the correct `firebase_options.dart` file
- Update your Android and iOS configuration files

### Step 5: Enable Required Firebase Services
In Firebase Console:
1. **Authentication**: 
   - Go to Build → Authentication
   - Click "Get Started"
   - Enable Email/Password and Google sign-in

2. **Firestore Database**:
   - Go to Build → Firestore Database
   - Click "Create database"
   - Choose "Start in test mode" (for development)
   - Select a location close to your users
   - Click "Enable"

3. **Storage** (for images/files):
   - Go to Build → Storage
   - Click "Get Started"
   - Start in test mode

4. **Cloud Messaging** (for notifications):
   - Already enabled by default

### Step 6: Set Up Security Rules
For development, use these Firestore rules (go to Firestore → Rules):
```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if request.auth != null;
    }
  }
}
```

### Step 7: Create Your First Admin User
After setting up Firebase:

**Option A: Via Firebase Console**
1. Go to Authentication → Users
2. Click "Add user"
3. Enter email and password
4. Go to Firestore Database
5. Create a collection called `users`
6. Add a document with the user's UID as the document ID
7. Add fields:
   - `email`: user's email
   - `role`: "admin"
   - `name`: user's name
   - `createdAt`: timestamp

**Option B: Via Code (Temporary)**
Modify the app to create an admin on first signup, then remove this code later.

### Step 8: Run Your App
```bash
flutter run
```

## Important Notes
- ⚠️ **DO NOT commit** `google-services.json` or `GoogleService-Info.plist` to Git if they contain sensitive keys
- For production, update Firestore security rules to be more restrictive
- Replace the placeholder `firebase_options.dart` I created with the one generated by `flutterfire configure`

## Troubleshooting
If you see errors about missing Firebase options:
1. Make sure you ran `flutterfire configure`
2. Check that `firebase_options.dart` exists in `lib/` folder
3. Verify the import in `main.dart` is correct

Need help? Run:
```bash
flutterfire configure
```
And follow the interactive prompts!
